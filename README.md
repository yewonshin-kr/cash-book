# 가계부 (v0.17)

개인용 가계부 웹앱. 구글 드라이브에 데이터를 자동 동기화하고, PWA로 설치해 실제 앱처럼 사용할 수 있습니다.

지출 기록·카테고리별 예산, 월별 결산, 저축·투자(IRP/ISA) 현황, 인세 정산 관리 기능을 제공합니다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `index.html` | 앱 본체 (단일 파일) |
| `manifest.webmanifest` | PWA 설치 정보 (앱 이름·아이콘·테마) |
| `sw.js` | 서비스 워커 (오프라인 캐시, 네트워크 우선) |
| `icon-192.png` / `icon-512.png` | 앱 아이콘 |

5개 파일을 **같은 폴더**에 두고 배포합니다. 별도 서버나 Apps Script 배포는 필요 없습니다.

## 배포 (GitHub Pages)

1. 저장소에 위 5개 파일 업로드
2. Settings → Pages에서 배포 활성화
3. `https://<아이디>.github.io/<저장소>/` 접속

> PWA 설치와 드라이브 연동 모두 **HTTPS 필수**입니다. GitHub Pages는 기본 HTTPS라 추가 설정이 없습니다.

## 데이터 저장 구조

- **기본 저장소: localStorage** (`bud_v1`) — 연결 없이도 브라우저에 데이터가 유지됩니다.
- **구글 드라이브 동기화 (선택)** — 연결하면 내 드라이브의 `my-budget-data.json` 파일 하나에 자동 저장되고, 다른 기기에서도 이어서 쓸 수 있습니다.
  - 변경 후 1.5초 뒤 자동 업로드 (디바운스)
  - 기기 간 충돌은 `updatedAt` 비교로 최신본 채택
  - 권한은 `drive.file` 스코프만 사용 — 이 앱이 만든 파일 외에는 접근하지 않습니다

## 구글 드라이브 연동 설정

앱 내: **설정 → 구글 드라이브 동기화 → 구글 계정 연결** 버튼 한 번이면 됩니다.

단, OAuth 클라이언트 ID를 Daily Hub와 공유하므로 **배포 주소(origin)가 클라이언트 ID에 등록**되어 있어야 합니다. Daily Hub와 같은 도메인(같은 `<아이디>.github.io`)에 배포하면 추가 설정이 필요 없습니다.

다른 도메인에 배포하는 경우:

1. [Google Cloud Console](https://console.cloud.google.com) → 클라이언트 ID를 만든 프로젝트 선택
2. **API 및 서비스 → 사용자 인증 정보** (또는 **Google Auth Platform → 클라이언트**)
3. 해당 OAuth 2.0 클라이언트 ID 수정
4. **승인된 JavaScript 원본**에 배포 주소 추가 — 경로 없이 도메인까지만 (예: `https://example.github.io`)
5. 저장 (반영까지 몇 분 소요)

`Error 400: origin_mismatch`가 뜨면 등록한 원본과 실제 접속 주소가 다른 것입니다.

### 토큰 동작 방식

- 액세스 토큰은 약 1시간 유효하며 localStorage에 캐시됩니다.
- 만료 10분 전부터는 화면 터치 시점에 무음으로 자동 재발급되어 재로그인 팝업 없이 이어집니다.
- 연결 해제는 설정에서 언제든 가능하며, 해제 시 토큰이 폐기됩니다.

## 앱 설치 (PWA)

- **Android Chrome**: 접속 → 메뉴(⋮) → **앱 설치** (또는 주소창의 설치 아이콘)
- **iOS Safari**: 공유 → **홈 화면에 추가**

설치하면 주소창 없는 독립 앱으로 실행되고, 오프라인에서도 열립니다.

## 업데이트 배포 시

- `index.html`만 수정한 경우: 그대로 재배포하면 다음 실행 때 자동 반영됩니다 (네트워크 우선 캐시).
- 아이콘·manifest를 바꾼 경우: `sw.js`의 캐시 이름 `budget-v1`을 `budget-v2`로 올려 캐시를 갱신하세요.

## 데이터 백업 · 이전

- 앱 내 **설정 → 데이터 내보내기**로 전체 지출 CSV를 받을 수 있습니다.
- 이전 버전(구글 시트 + Apps Script 방식)의 시트 데이터는 자동으로 이전되지 않습니다. 시트에 코드 내장분보다 최신 데이터가 있다면 CSV 가져오기로 옮겨주세요.
